# Admin Fallback Notes

Use fallback mode when the public app, internet connection, QR flow, login, proof submission, admin review, hosting provider, or production state is unreliable.

Manual loop:

1. Find the quest in `qr-mapping.md`.
2. Read or send the matching quest from `quests.md`.
3. Receive proof outside the app through a chat app, AirDrop, or cloud link.
4. Approve or reject manually.
5. Update `team-progress.md`.
6. Reveal the next fragment from `fragments/` or the manual reveal method.
7. At 21 approvals, show `final-prize-photo.jpg`.

Before the event:

- Replace `https://mission.example` in `qr-mapping.md` with the final production URL.
- Fill QR hiding locations.
- Replace placeholder quest copy if final event copy changes.
- Add the real full map image as `map-full.png`.
- Add fragment files under `fragments/` or document the manual reveal method in `map-reveal-method.md`.
